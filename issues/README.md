# Issues Directory

> Folder ini berisi file individual untuk setiap issue yang harus dikerjakan. Setiap file berisi detail task, acceptance criteria, dan implementasi notes.

## Struktur Penamaan

Format: `NNN-issue-title.md` (contoh: `007-database-schema.md`)

## Status

Semua file issue akan dibuat secara bertahap sesuai kebutuhan. Untuk melihat daftar lengkap issue dan statusnya, lihat [ISSUES_INDEX.md](../ISSUES_INDEX.md) di root repo.

## Cara Menggunakan

1. Pilih issue berikutnya yang harus dikerjakan dari `ISSUES_INDEX.md`
2. Baca file issue yang sesuai di folder ini (jika sudah ada)
3. Kerjakan sesuai acceptance criteria yang tercantum
4. Update status di `ISSUES_INDEX.md` setelah selesai
5. Commit & push perubahan

## Catatan

- File issue bisa dibuat on-demand saat akan dikerjakan
- Tidak semua issue perlu file terpisah — issue sederhana cukup mengacu ke dokumen utama (PRD.md, DATABASE.md, dll)
- Issue yang kompleks (misal: checkout flow, payment integration) sebaiknya punya file detail tersendiri
